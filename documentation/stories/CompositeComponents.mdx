  <p>Composite components are built by combining simpler components to create more complex structures. They allow for greater flexibility and reusability in UI design.</p>

<h2>Examples of Composite Components</h2>
<p>
  Here are some examples of composite components you can create using the KNIME
  Design System:
</p>
<ul>
  <li>
    Card: A component that combines an image, title, description, and action
    buttons.
  </li>
  <li>
    Form: A composite that includes input fields, labels, and buttons for user
    interaction.
  </li>
  <li>
    Navigation Bar: A component that combines links and dropdowns for site
    navigation.
  </li>
</ul>

  <h2>Best Practices</h2>
  <p>When creating composite components, consider the following best practices:</p>
  <ul>
    <li>Ensure that the composite component is accessible and follows best practices for usability.</li>
    <li>Keep the component modular to allow for easy updates and maintenance.</li>
    <li>Document the properties and usage examples for each composite component.</li>
